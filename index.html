What is the marketing bundle?

A library of 619 AI skills tailored to Domo marketing work

The bundle lives on your laptop and stays current with a 15-minute auto-sync. You'll discover skills that match your team — Communications, Digital, Creative, Product Marketing, Integrated, Field, Experiential, Analytics, CMO, EA — and use Claude Code to run them. We'll cover all of that in our onboarding meeting. This page just walks you through the five things to get done before then.

Get Claude Code access

Domo has a Claude Team plan. John Brandt manages access. Send him a quick note and he'll add your @domo.com email to the team. You'll log into Claude.ai with your Domo email and sign in via OKTA when approved.

What is Claude Code? An AI assistant that runs inside VS Code. It's your pair-programmer for marketing work — drafting copy, building landing pages, querying data, generating images. You won't be writing code yourself.
2
1-3 business days

Request your Domo GitHub account

The bundle lives in a private Domo GitHub repo. You'll need a Domo Enterprise Managed User (EMU) account to access it. Request one through Domo ServiceNow — IT typically approves within 1-3 business days, and you'll get a GitHub email to set your username.

Open ServiceNow request
Don't skip this. Without a Domo GitHub account, you can't access the bundle repo. Steps 1 + 2 run in parallel — kick both off now so the approvals come in while you finish the rest.
When approved, GitHub will email you to set your username. Your username will end with _domo (e.g., cory.edwards_domo). Save it — you'll use it in Step 5.
3
~5 minutes

Download VS Code

Claude Code runs inside VS Code — Microsoft's free code editor. Don't worry, you won't need to write any code. This is just where Claude lives.

Download VS Code
You'll need admin access on your Mac. Click the check mark in your top bar, request administrator access, reason: "VS Code install." Domo IT usually grants it within minutes.
4
~5 minutes (after Step 1 is approved)

Install the Claude Code extension

Once VS Code is installed and your Claude Team email has arrived (Step 1), add the Claude Code extension and sign in.

  1. Open VS Code
  2. Click the Extensions icon in the left sidebar (or press Cmd + Shift + X)
  3. Search for "Claude Code"
  4. Click Install on the official Anthropic extension
  5. Click the Claude icon in the sidebar to open it
  6. Click Sign In and use your @domo.com email (the one John added to the Claude Team plan)
Smoke test: Click the Claude icon → type "hi" → confirm you get a friendly response. If you do, the extension is working and you're signed in correctly.
5
~10 minutes (after Step 2 is approved)

Connect GitHub

After your GitHub account is provisioned (Step 2), authenticate the GitHub CLI on your laptop. This lets Claude Code clone the bundle and open pull requests on your behalf.

Open the Terminal in VS Code (View → Terminal, or press Cmd + `) and paste each block in order.

a. Make sure the GitHub CLI is installed

# If `gh --version` errors out, install with Homebrew: brew install gh

b. Authenticate (a browser window will open)

gh auth login

When prompted, choose: GitHub.com → HTTPS → Yes (authenticate Git) → Login with a web browser. Sign in with your @domo.com EMU account from Step 2.

c. Verify it worked

gh api user --jq .login

You should see your username — it will end in _domo.

"User not found"? Your GitHub provisioning may not be fully complete yet. Wait an hour and re-run the verify step. If it still fails after a few hours, ping Jake.
Don't have brew? Install it from brew.sh first — it's the macOS package manager and you'll use it for everything else too.
